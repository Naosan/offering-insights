Privacy Policy

This policy explains what information Offering Insights accesses, collects, processes, stores, shares, refreshes, and deletes. The API Client owner is Naoki Ishibashi and privacy questions can be sent to offeringinsights.contact@gmail.com.

Current service scope: selected public YouTube metadata only. Offering Insights does not provide Google OAuth sign-in, access private YouTube account data, request Authorized Data, upload or modify YouTube content, or manage accounts, channels, comments, or credentials.

Information we access, collect, store, and use

The following table distinguishes user-provided planning information, metadata request inputs, public YouTube API Data, technical request information, and contact messages.

Data category What is accessed or collected How it is used and shared Storage, retention, and deletion
Selected-source input Public YouTube video IDs, public playlist IDs parsed from URLs supplied by the user, and the selected category region (JP, US, or GB). The browser sends only these parsed IDs and the region to the Offering Insights managed service. The service sends them to YouTube Data API Services as read-only request parameters to retrieve public metadata. Not written to an application database, server-side cache, or application log. The input exists only while the live request is processed and is not retained by the application after the response completes.
Planning input The note name, use case, review question, and planning note entered in the workspace. Used only by browser code to format the user-authored source review note. These fields are not sent to the managed service, Google, or YouTube. Held only in the current page state and removed from application state when the page is reloaded or closed. A copy persists only when the user chooses to copy or download it locally.
Public YouTube API Data Public video, channel, category, and optional playlist-item metadata returned by videos.list, channels.list, videoCategories.list, and playlistItems.list. This can include IDs, titles, channel names, descriptions, category labels, thumbnails, publication dates, duration, public status, and public statistics such as views, likes, comments, subscriber counts, and video counts. Used to show each selected YouTube source, its public category label, its public channel context, and the API trace in the requesting user's browser. The application does not aggregate categories, calculate representation, rank or score sources, identify trends, or generate recommendations from API Data. The data flows from YouTube to the managed service and then to that browser. Offering Insights does not sell it or use it for advertising or user profiling. Not stored in an application database, server-side cache, or application log. API responses are returned with Cache-Control: no-store, kept only in the current page state, replaced by the next lookup, and removed from application state when the page is reloaded or closed.
Technical request information Standard web request information may include source IP address, user agent, timestamp, request method and path, response status, and latency. Used by Google Cloud to operate and secure the managed service. The application derives an instance-specific HMAC value from the source address for abuse rate limiting; the raw address is not stored in the application rate-limit map. The HMAC rate-limit entry is held in process memory for no more than 10 minutes. Google Cloud's standard request logs are retained for 30 days. The application does not write metadata request bodies, selected source IDs, or YouTube API responses to those logs.
Contact information An email address and any information a person chooses to include when contacting Offering Insights. Used only to answer support, privacy, deletion, or policy questions and processed through Gmail. Retained only as needed to resolve the request or maintain required compliance records. Personal information under our control will be deleted within 7 calendar days after a valid deletion request unless retention is legally required.

API Project and credential boundary

Offering Insights uses exactly one Google Cloud API Project for this API Client: project number 141682939002. No other project number is used for this API Client. The project API credential is stored in Google Secret Manager and used only by the managed Cloud Run service. It is never entered by, returned to, issued to, or stored in a public user's browser, and no credential is generated per user or per metadata request.

API Data refresh, update, and deletion

  • Every metadata lookup makes new read-only YouTube Data API requests and displays the most recently returned public API Data. Offering Insights does not reuse a stored API response.
  • Running another lookup replaces the prior result in the current page with newly retrieved API Data.
  • Reloading or closing the page removes the live result from Offering Insights application state.
  • The managed service does not persist API Data after completing the request and does not maintain an API Data database or cache.
  • Because Offering Insights does not retain API Data, it does not keep Non-Authorized Data for 30 days. If persistent API Data storage is introduced later, this policy and user consent flow will be updated before that use begins, and stored data will be refreshed or deleted no later than the period required by the YouTube API Services Developer Policies.
  • A report copied or downloaded at the user's direction is stored only in the user's clipboard or device. The user controls and deletes that local copy.

Service providers and data sharing

  • Google and YouTube: receive the selected public IDs, category region, and standard request information needed to provide YouTube Data API Services and public thumbnail images. Their handling is governed by the Google Privacy Policy.
  • Google Cloud: hosts the managed service, stores the project credential, and retains the standard operational request logs described above.
  • GitHub Pages: hosts the public static interface and may process standard web request information under the GitHub General Privacy Statement.
  • jsDelivr: serves a fixed Three.js library file used for the optional evidence map. The library request does not include selected YouTube IDs or returned API Data, but jsDelivr may receive standard web request information such as an IP address and user agent.
  • Gmail: processes messages sent to the Offering Insights contact address.

Offering Insights does not share API Data or user information with advertising networks, data brokers, or unrelated third parties. It does not allow third parties to serve advertisements.

Cookies and information on the user's device

Offering Insights application code does not set advertising or analytics cookies and does not use localStorage or sessionStorage for metadata request inputs or API Data. Live results remain in page memory. A file or clipboard copy is created only after the user selects the corresponding export action.

Security

Data is transmitted over HTTPS. The managed service restricts accepted browser origins, limits request size and frequency, keeps the API credential in managed secret storage, and returns API responses with no-store instructions. Human access to operational systems is limited to maintenance, security, compliance, and support needs.

Deletion requests and user choices

To request deletion of personal information controlled by Offering Insights, email offeringinsights.contact@gmail.com with the subject "Data deletion request." We will respond and delete identifiable personal information under our control as soon as possible and within 7 calendar days, unless retention is legally required. Offering Insights does not maintain user accounts or a persistent API Data store, so live source inputs and API results normally clear automatically as described above.

Deleting information held by Offering Insights does not delete data held by YouTube. A person who wants to delete or change information on YouTube must use YouTube or an authorized API Client that supports that action.

Google account permissions

This version does not request Google OAuth access, User Credentials, or Authorized Data. If a future version introduces Google authorization, users will be asked to accept an updated privacy policy before that access begins and will be able to review or revoke access through Google's security settings page.
