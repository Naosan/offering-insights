Privacy Policy

Offering Insights is a prototype video research dashboard. This policy explains how the service handles information when using YouTube Data API Services.

Current prototype scope: public metadata lookup only, no Google OAuth sign-in, no private YouTube account data, and no upload or account-management functions.

Information we collect

The temporary public website does not require account registration, does not use advertising cookies, and does not ask visitors to sign in with Google. If you contact us by email, we receive the information you choose to send, such as your email address and message.

YouTube Data API Services

The API client uses YouTube Data API Services to retrieve public metadata such as video titles, video IDs, channel IDs, category information, playlist item metadata, publication dates, and public statistics. The current version does not request Google OAuth access and does not access private YouTube account data.

Visitors provide public YouTube video IDs, video URLs, or public playlist URLs for analysis. The browser sends the parsed public IDs and selected category region to the Offering Insights managed service. That service makes the approved read-only requests to YouTube Data API Services. The project API credential is stored in managed server configuration and is never entered into, returned to, or stored by the visitor's browser.

Use of YouTube Data API Services is also governed by the YouTube Terms of Service, the YouTube API Services Terms of Service, and the Google Privacy Policy.

How we use information

The public source IDs parsed from visitor-supplied URLs, the selected category region, and the required read-only request parameters are processed by the managed service and transmitted to Google API endpoints. Returned public YouTube metadata is sent to the current browser session to organize the selected source set, compare public channel context, and prepare editorial, category, or research notes. We do not sell personal information. We do not use Google user data for advertising targeting. Human access to operational data is limited to maintenance, security, policy, or user support needs.

Storage and deletion

The current service does not persist YouTube API response data or visitor source lists in an application database. Live results remain in the current browser session, and a report is saved locally only when the visitor chooses the copy or download action. The managed service may retain short-lived operational logs needed for security, availability, and abuse prevention; those logs are not used to build user profiles. If caching is added later, cached public API data will be limited to what is needed for product functionality and will be refreshed or deleted within the applicable YouTube policy period. Requests for deletion or correction can be sent to offeringinsights.contact@gmail.com.

Google account permissions

This version does not request Google OAuth access or Authorized Data. Google account permissions can be reviewed or revoked at any time from Google's security settings page.

Data sharing

Public source IDs and the request parameters needed for the selected analysis are transmitted to Google through YouTube Data API Services. We do not transfer Google user data to other third parties except as required for operating the service, complying with law, security review, or with the user's explicit direction. The current prototype does not collect Google OAuth user data.

Contact

For privacy questions, data deletion requests, or policy questions, contact offeringinsights.contact@gmail.com.
